# System Overview

## 1. 系统定位

这个系统的目标不是做一个“会聊天的助手”，而是做一个 **可执行的个人管理框架**。

核心作用：

- 收集信息
- 结构化整理
- 排定优先级
- 建立提醒
- 跟踪执行
- 形成闭环

## 2. 管理板块

### A. 学习
学习板块分为两条线：

#### A1. 学校学习
- 课程 assessment
- 考试
- DDL
- tutorial / workshop 出勤
- group project
- final exam / hurdle requirement

#### A2. 自学
- 独立学习计划
- 技能补充
- 假期专项学习
- 长期能力建设

### B. 生活
后续可纳入：
- 作息
- 睡眠
- 日常事务
- 健康
- 金钱与消费
- 行政事项

### C. 休闲
后续可纳入：
- 社交
- 娱乐
- 运动
- 兴趣探索
- 放松安排

## 3. 当前推进方式

当前采取 **从考试和 assessment 开始** 的方式推进，原因是：

- 时间节点最清晰
- 风险最直接
- 最适合建立提醒与执行闭环

也就是说，系统当前不是一次性覆盖所有生活内容，而是先从 **最能量化、最能执行、最容易出结果** 的板块建立。

## 4. 管理逻辑

每个事项按照以下顺序处理：

1. **收集信息**  
   先确认课程代码、assessment 组成、占比、DDL、exam date、hurdle requirement

2. **判断优先级**  
   根据以下因素排序：
   - 时间是否接近
   - 占比是否高
   - 容错是否低
   - 是否需要持续投入

3. **制定执行方案**  
   把模糊任务改写为：
   - 什么时候做
   - 做什么
   - 做到什么算完成

4. **设置提醒**  
   不依靠临时记忆，而是用提醒固定关键节点

5. **回报结果**  
   做完后需要回报状态，形成闭环

## 5. 执行原则

### 原则 1：不接受模糊状态
例如：
- “差不多”
- “应该没问题”
- “之后再说”
- “以前学过”

这些都不是可执行信息。

### 原则 2：优先处理高权重 + 低容错事项
典型例子：
- 高占比作业
- hurdle final exam
- 出勤要求明确的课
- 近期考试

### 原则 3：持续低强度胜过考前堆积
对于考试课，优先采用：
- 每周固定复习
- 近考前集中强化
- 练习驱动，而不是只看笔记

### 原则 4：任务必须形成闭环
一个完整任务不只是“提醒去做”，还包括：
- 做完
- 提交 / 执行
- 回来汇报
