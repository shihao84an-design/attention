# Reminders and Rules

## 1. 提醒系统说明

提醒的用途不是“提供心理安慰”，而是为了把关键节点变成固定动作。

当前采用的是 ChatGPT 任务提醒机制，适用于：
- 一次性提醒
- 阶段性提醒
- 周期性提醒
- 执行后回报提醒

## 3. 提醒上限处理方式

当前提醒系统存在 **活跃提醒数量上限（10 个）**。

因此后续增加提醒时，优先使用以下策略：

1. **合并提醒内容**  
   把多个动作写进一个提醒，而不是拆成很多碎提醒

2. **删除低价值提醒**  
   如果提醒失去作用，应及时移除，为更重要事项腾位置

3. **保留关键节点提醒**  
   例如：
   - 提交前
   - 考试前
   - 固定出勤前
   - 回报闭环提醒

## 4. 执行规则

### 规则 1：提醒不是完成
收到提醒不等于任务完成。真正完成必须满足：
- 已执行
- 有结果
- 能汇报状态

### 规则 2：考试课学习必须包含练习
不能只看 slides 或笔记。最低标准是：
1. 看一个知识点
2. 做对应练习
3. 标记错误或薄弱点
4. 下次优先补弱项

### 规则 3：高风险出勤课默认必去
目前默认必须保证出勤的课程：
- ACCT1006 tutorial
- BUSS2000 workshop
- IBUS2101 tutorial / class（当前参与度已有压力）

### 规则 4：每项高风险任务要形成闭环
标准流程：
- 提前提醒
- 执行
- 提交 / 完成
- 回报结果
